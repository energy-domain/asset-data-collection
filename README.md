# Asset Data Collection  
**Repository for the LOIN Phase**: Structured raw data collection of energy assets (properties, metadata) before ontological modeling.  

## Purpose  
This repository stores:  
- ✅ **Inventories of assets** per subdomain (e.g., `subsea-flexible-pipes`, `nuclear-reactors`).  
- 📊 **Properties and metadata** (data types, sources, units) for each asset.  
- 🔍 **Expert-reviewed datasets** (pre-ontology validation).  

Designed to support:  
- Current workflow: **BIM/IFC specifications**.  
- Future expansion: **Other standards (GIS, ISO 15926, etc.)**.
